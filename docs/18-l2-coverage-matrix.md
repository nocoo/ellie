## 2. Parser contract

Routes are extracted from `apps/worker/src/index.ts` using three
regex shapes:

```
if (path === "<literal>" && request.method === "<METHOD>") {
if (path.match(/<regex>/) && request.method === "<METHOD>") {
const m = path.match(/<regex>/);
if (m && request.method === "<METHOD>") {
```

Method-less guards (e.g. CORS preflight `OPTIONS`, the
validateApiKey middleware, the maintenance gate) are intentionally
**not** counted as routes.

L2 calls are recognized in two forms:

1. **Helper calls** ‚Äî the helper map declared at the top of the
   script (mirrors `tests/integration/setup.ts`):

   - `workerFetch` ‚Üí GET
   - `workerAuthFetch` ‚Üí GET
   - `workerPost` ‚Üí POST
   - `workerPatch` ‚Üí PATCH
   - `workerDelete` ‚Üí DELETE
   - `adminFetch` ‚Üí GET
   - `adminGet` ‚Üí GET
   - `adminPost` ‚Üí POST
   - `adminPatch` ‚Üí PATCH
   - `adminPut` ‚Üí PUT
   - `adminDelete` ‚Üí DELETE

2. **Raw `fetch(...)` to the Worker** ‚Äî calls of the form
   `fetch("http://localhost:8787/api/...", init?)` or
   `fetch(`${getWorkerUrl()}/api/...`, init?)` (also
   `${WORKER_URL}`). Default method is `GET`; an `init` object with
   `method: "POST" | "PATCH" | ...` overrides it. Only the literal
   `localhost:8787` prefix and the two known template helpers are
   recognized ‚Äî external URLs and non-Worker `fetch` calls are
   ignored. Scanning for raw fetches is restricted to
   `tests/integration/worker/**` to avoid false positives.

Test paths with `${...}` template substitutions are rewritten to
`:param` and matched against literal routes verbatim or against
regex routes by substituting `:param` with `1` (numeric path
params) or `abc` (string path params), in that order.
